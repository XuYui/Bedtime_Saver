# 更新日志

本项目所有修正、升级和开发规范调整都必须记录在这里。最新条目放在最顶部。

## 1.2.2 - 2026-05-23

### 修复

- 移除 App 内自建晨起闹钟功能，恢复为用户使用系统闹钟。
- 睡眠监督状态下放行常见系统时钟/闹钟应用，避免闹钟响起或关闭时被阻断页拦截。
- 清理晨起闹钟设置、响铃页、闹钟 Receiver 和震动权限。

## 1.2.1 - 2026-05-23

### 修复

- 停止在目标就寝时间自动开启睡眠监督，监督模式现在仅在用户点击「我要睡了」后开启。
- 兼容取消旧版本可能已调度的目标就寝时间监督闹钟，避免覆盖安装后仍被自动拉起监督。
- 确认睡眠监督状态下 App 自身不会被拦截，用户仍可打开 App 点击「我起床了」。

## 1.2.0 - 2026-05-22

### 新增

- 新增晨起闹钟设置，用户点击「我要睡了」后会按设定时间调度系统闹钟。
- 新增全屏闹钟响铃页，支持唤醒屏幕、循环铃声和震动提醒。

### 修复

- 修复睡前打卡后没有真实晨起闹钟响铃链路的问题。
- 闹钟关闭后会自动完成晨起打卡、计算睡眠时长，并退出睡眠监督状态。

## 1.1.0 - 2026-05-21

### 新增

- 新增补充打卡功能，可手动录入睡眠日期、入睡时间和起床时间，并自动计算睡眠时长、达标状态和连续天数。
- 新增睡眠质量评测标准说明，明确 A+、A、B、C 的判定逻辑。

### 优化

- 将「近35日表现」改为本月打卡日历，历史记录仍长期保存在本地数据库。
- 移除目标就寝时间卡片中的建议洗漱时间文案。

## 1.0.1 - 2026-05-20

### 修复

- 修复底部导航「今晚」标签被月亮图标挤压，导致文字底部溢出或裁切的问题。

## Docs - 2026-05-20

### 优化

- 将 `DEVELOP.md` 重写为面向后续 AI 开发者的接手指南，明确项目定位、技术栈、UI 设计轨道、数据规则、验证规范和最终回复规范。
- 新增“每次修正或升级必须同步 GitHub，并上传版本更新说明”的流程要求。
- 新增 `CHANGELOG.md`，作为后续所有版本说明和 GitHub 同步的唯一来源。

## 1.0.0 - 2026-05-20

### 新增

- 完成 Bedtime Saver 首个可运行版本：睡前打卡、晨起打卡、连续早睡统计、近 35 日记录点阵。
- 实现无障碍前台应用检测、全屏睡眠监督阻断页和 60 秒清醒挑战。
- 提供深色夜间视觉设计、Stitch 设计稿、README、发行 APK 和 GitHub Release。

### 优化

- 修复 edge-to-edge 模式下底部导航可能被系统导航栏遮挡的问题。
- 替换应用启动图标，并刷新 `release/BedtimeSaver-v1.0.0.apk`。
